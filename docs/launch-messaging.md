# LARC Launch Messaging

> Recommended messaging for the initial docs-first open-source opening.

---

## 1. Messaging Goal

The goal of the first public opening is:

- to open a credible public door
- to explain the project wedge clearly
- to avoid overclaiming runtime completeness

The messaging should make it obvious that:

- LARC is real
- LARC is technically interesting
- LARC is still in incubation

---

## 2. Core Positioning

Use this idea consistently:

**LARC is a permission-first runtime for Lark-native office-work agents.**

Avoid framing it as:

- a finished enterprise platform
- a generic automation toolkit
- a coding-agent-only project
- a fully frozen runtime release

---

## 3. Short GitHub Repository Description

Recommended:

`Permission-first runtime for Lark-native office-work agents.`

Alternatives:

- `Lark-native runtime for office-work agents, focused on permissions, authority, and execution control.`
- `Open-source Lark agent runtime for office workflows, with permission-first design.`

Avoid:

- `Best Lark agent platform`
- `Production-ready enterprise agent OS`
- `Fully autonomous office agent system`

---

## 4. README Opening Guidance

Recommended current stance:

- keep the title as `LARC — Lark Agent Runtime`
- keep the tagline short
- mention office-work agents before mentioning CLI mechanics
- mention incubation before claiming maturity

Recommended opening shape:

1. one-line positioning
2. what problem it solves
3. what is already real
4. what is still incubating

---

## 5. Initial Public Opening Note

### English

`LARC is now opening as a docs-first open-source project.`

`The project explores a permission-first runtime for Lark-native office-work agents, with Lark Drive, Base, IM, Approval, and Wiki treated as the operating surface rather than external integrations.`

`This initial opening focuses on the project story, permission model, trilingual documentation, and public contribution surface. The runtime implementation is real, but still being frozen into a cleaner public release slice.`

### Japanese

`LARC を docs-first のオープンソースとして公開します。`

`このプロジェクトは、Lark Drive・Base・IM・Approval・Wiki を単なる連携先ではなく実行面として扱う、permission-first な一般業務エージェント runtime を探るものです。`

`今回の初回公開では、プロジェクトの意図、権限モデル、三言語ドキュメント、公開向けの参加導線を先に開きます。runtime 実装は存在していますが、よりきれいな公開スライスに整理している途中です。`

### Chinese

`LARC 现在以 docs-first 的方式开启首次开源。`

`这个项目探索的是一种面向 Lark 原生办公 Agent 的 permission-first runtime，把 Drive、Base、IM、Approval、Wiki 视为运行表面，而不是单纯的外部集成。`

`这次首次公开主要开放项目定位、权限模型、三语言文档与公共贡献入口。runtime 实现并非不存在，但仍在整理为更清晰的公开切片。`

---

## 6. What To Emphasize

- permission-first design
- authority clarity
- Lark-native operating surface
- office-work agent focus
- China-facing open-source intent

---

## 7. What To Downplay

- any implication that the full runtime is already frozen
- any implication that the project is feature-complete
- any implication that all legacy assets are part of the primary public surface

---

## 8. Recommended Release Tone

Target tone:

- calm
- technically honest
- ambitious but not inflated
- specific about what is real today

If the wording sounds more advanced than the current Bundle A readiness, simplify it.
